C++ Foundation


C++ Foundation is my simple C++ library providing many useful classes from string manipulation to TCP/IP communication. As a lightweight library Written in raw C++, it doesn't use any of 3rd party libraries only C++ Standard library, Win32 API and Linux System Calls. C++ Foundation can be compiled for both Windows and Linux platforms.

C++ Foundation was designed to simplify application development on small embedded systems that can't afford to host much more robust and more resource consuming frameworks like QT or Boost.
C++ Foundation is written in raw C++ with a small portion of code written in native OS libraries Win32 API and Linux System Calls to implement OS specific features that are not provided by C++ Standard library. The library can be compiled by Visual Studio for Windows platform and by G++ 8 for Linux platform.

The following table shows the list of most important features.

Feature Description
Collections Mostly wrappers around the C++ Standard Library collections. It provides easier to use API above STL collections and many useful methods that simplify the development. There is also the class CByteBuffer for easy byte array manipulations.
Communication The library for TCP/IP, UDP and CAN bus communication. Since C++ Standard Library for C++ 11 doesn't provide network communication facilities, I had to implement platform specific code for both Windows and Linux platforms. The library uses asynchronous API that doesn't block more then 1 thread for up to 50 communication channels.
Console A simple helper for working with console. Class CConsoleMenu allows you to create a simple console application with text menu that is very helpful if you need to create small testing console programs. I used it very often in my projects.
Exceptions A bunch of simple exception classes. C++ Foundation uses exception handling for error propagation instead of return values or other means.
FileSystem Once again a platform specific library that allows you to work with files and directories on Windows and Linux platforms. At the time of writing C++ didn't offer any elegant way to work with the file system besides C functions and rather cumbersome C++ Stream. So, I wrote classes CDirectory, CFile and CFilePath to make the things easier.
Logger A simple console logger.
Objects The only class here CObject is a base class for all other classes in this library.
Parsing A helpful library for the byte array parsing.
StateMachines I needed to use state machine pattern a lot, so I developed this library allowing you to work with the state machines. A state machine can create subordinate state machines and wait until their job is done, so you can create a tree of interconnected state machines. State machines are a useful tool for low level programming where state machine pattern is used quite often.
Strings The core class is CString that provides many functionalities missing in C++ Standard Library class std::string. There is also a very useful method called Format() that mimics the functionality of C# String.Format() function using variadic templates.
Threads Class CThread implements a producer-consumer pattern around C++ Standard Library class std::thread. There are also other wrappers around C++ Standard Library threading facilities providing features like locks, mutexes, auto reset events and manual reset events.
Time The core class is CTime that provides similar functionalities like C# DateTime class.
Types The most important class is CSharedPointer a wrapper around C++ Standard Library std::shared_ptr an extremely useful class doing reference counting for naked pointers that prevents all problems with the memory leaks. I use this class instead of naked pointers whenever possible. There is a small overhead but it saves you a lot of development effort.
