IT BOOKS


This page contains some of IT books I have studied during my 20+ year career as a programmer. When studying IT books I always try to create useful code samples that serve me as a cookbook when I need to use or understand some feature later. All of these code samples are my personal work, not just downloaded samples from associated book web pages. The comments and remarks are written in my native language Slovak. All source codes are available on the GITHUB.

Advanced .NET Debugging

Author: Mario Hewardt
Publisher: Addison-Wesley Professional
Publication date: 9 November 2009
Pages: 542
Review: A quite interesting book about advanced debugging techniques in .NET that go far beyond 'standard' debugging tools in Visual Studio. Normally you don't need to use these techniques but in some cases like post mortem debugging, doing COM interoperability, or having problems with the memory (Large Object Heap is not defragmented in .NET), techniques presented in this book are being very helpful.

Applications = Code + Markup: A Guide to the Microsoft Windows Presentation Foundation

Author: Charles Petzold
Publisher: Microsoft Press
Publication date: 13 September 2006
Pages: 1020
Review: The book about WPF from the legendary author Charles Petzold well known for his Win32 API programming book. The book provides unique way explaining WPF. The first half of the book is all about C# code. No XAML at all. It shows you the WPF programming model that is independent to XAML. The second part of the book is all about XMAL. It teaches you how to create static and dynamic forms only in XAML. Since WPF is a part of .NET Core, it's an evolving technology that allows you to create non-web GUI applications under Windows, Linux or MacOS.

C++ Templates: The Complete Guide, 2nd Edition

Authors: David Vandevoorde, Nicolai Josuttis, Douglas Gregor
Publisher: Addison-Wesley Professional
Publication date: 8 September 2017
Pages: 832
Review: The best book about a powerful but rather complicated topic - C++ templates. The book goes very deep into C++ templates. It's not an easy reading and it takes time and doing a lot of experiments to fully understand all traps and pitfalls of C++ templates. But since C++ templates is the strongest language feature that is regularly extended in every version of C++, I strongly recommend this book to each experienced C++ developer.

CLR via C#, 4th Edition

Author: Jeffrey Richter
Publisher: Microsoft Press
Publication date: 15 November 2012
Pages: 896
Review: CLR via C#, 4th Edition is by far the most comprehensive and deepest explanation how C# and CLR work. It gives you a lot of insights about the CLR internals. There is also a large section about multithreading and task programming with one of the best explanation of how thread, tasks and synchronization primitives work and when to use them. Jeffrey Richter as a seasoned expert in Windows technologies provides a unique, clear and easy to understand explanation of CLR and C#. It is a must read for every C# developer. The book was written for .NET Framework 4.0 but it is perfectly valid for .NET Core. I consider CLR via C# as the best book that has been written about the CLR and C#.

Expert .NET 2.0 IL Assembler

Author: Serge Lidin
Publisher: Springer
Publication date: 24 August 2006
Pages: 529
Review: Normally you don't need to know about MSIL and still being a successful .NET programmer. MSIL is underlying assembler language to which .NET languages like C# compile its code. Most .NET programmers have never ever written any code in MSIL since C# and other high level languages are much easier to use. But if you like to delve deep into technology and to understand how it works this book does a great job. CLR also offers some features that are not supported in C# and can be accessed only via MSIL. A newer edition of this book is available.

Windows Kernel Programming

Author: Pavel Yosifovich
Publisher: Independently published
Publication date: 26 February 2023
Pages: 625
Review: This kernel programming book is very unique. There are not many Windows kernel programming books on the market and most of them are very old ones. Windows Kernel Programming is written for Windows 10 with remarks about Windows 11. Explanations of this hard topic are easy to understand for Windows programmers. Samples are up to date with the latest Windows and Visual Studio so you can easily try them. The book is written for WDM API that sits bellow all other high level kernel programming APIs like WDF. It means you really look under the hood of the Windows. The only drawback is that book is purely written for SW drivers, which means that things like PNP, Power Management or access to the system bus are not covered here. But easy to understand and concise explanations make this book as an excellent starter for every Windows kernel developer.
I haven't finished the book yet. About 2/3 of the book.
